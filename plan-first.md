# Plan First, Code on Approval

For the following task, first provide a step-by-step plan: the files you will
change and the logic/approach for each. DO NOT write or edit any code yet.

Reading, searching, and inspecting the codebase to build an accurate plan is
allowed and encouraged. The gate applies to changes only: no file edits, no
builds, tests, or destructive/state-changing commands until I approve.

Plan format (keep it tight):
- Files to change (path + one line on what/why).
- Logic or approach per file.
- Order of changes and how you'll verify.
- Open questions or assumptions, if any.

After showing the plan, stop and wait:
- Affirmative (e.g. "go ahead", "go", "proceed", "yes", "do it", "approved",
  or any clear confirmation — match intent, not the exact word): implement the
  full plan.
- Scoped approval (e.g. "just do step 2"): implement only that part, then stop.
- Feedback or changes: revise the plan, re-show it, and wait again. Do not
  start coding on feedback alone.
